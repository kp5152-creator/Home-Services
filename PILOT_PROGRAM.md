# EstateIQ Pilot Program

## Purpose

Run focused customer pilots that prove EstateIQ can help luxury home watch, property management, and hospitality operators deliver clearer inspections, better homeowner communication, and stronger operational accountability.

## Recommended Pilot Structure

- Pilot length: 30 to 60 days.
- Pilot size: 2 to 5 organizations.
- Properties per pilot: 1 to 10 homes.
- Users per pilot: 1 admin, 1 to 3 inspectors, selected homeowners when appropriate.
- Weekly check-in: 20 minutes with the operator or property manager.
- Success review: compare before/after workflow time, report quality, homeowner confidence, and issue response.

## Onboarding Pilot Users

1. Create the pilot organization.
2. Add one pilot property with realistic homeowner/profile data.
3. Add users by role: Admin, Inspector, Homeowner, Property Manager.
4. Confirm demo-safe data is separate from real customer data.
5. Walk the user through Dashboard, Property, Inspection, Maintenance, Reports, and Owner Portal.
6. Have the inspector complete one guided first inspection.
7. Review the generated homeowner report and PDF together.
8. Capture feedback before ending the session.

## Feedback Collection

Use the in-app feedback panel for:

- Thumbs up/down after important workflows.
- Feature requests.
- Bug reports.
- Post-inspection survey notes.
- Homeowner satisfaction comments.

During calls, ask:

- What felt easiest?
- Where did you slow down?
- What would make this feel more premium?
- Would you send this report to a homeowner today?
- What would you pay for this if it saved your team time?

## Success Metrics

- Admin can create or select a property without help.
- Inspector can complete an inspection in the field.
- Photos upload and display clearly.
- PDF report is homeowner-ready.
- Maintenance issue can be documented with photos and vendor assignment.
- Homeowner understands property status within 30 seconds.
- Pilot customer asks to use it again after the demo.

## Operating Rules

- Do not expose real access codes in demo mode.
- Keep AI suggestions human-reviewed before homeowner sharing.
- Keep pilot scope narrow until customers validate the workflow.
- Track friction before adding features.
- Prioritize clarity, speed, and trust over complexity.
